# ntb_delivery
## Les élèves fournissent un carnet Jupyter fonctionnel

# ntb_intro	
## Les élèves présentent de manière pertinente leur cahier (aperçu, résumé, exigences, objectifs, etc.)

# ntb_q&a
## Au moins 3 questions pertinentes sont clairement définies et répondues

# ntb_format	
## Les élèves exportent leur carnet dans un autre format pour faciliter le partage

# ntb_clarity
## Le carnet contient des explications claires sur la méthodologie, les processus et le code

# ntb_summary	
## Les étudiants résument leurs résultats, indicateurs de performance et analyses dans un format multiplateforme (comme le PDF)

# data_visuals	
## Le carnet contient des visualisations de données lisibles (diagrammes, graphiques, etc.) bien présentées (titre + légende + étiquettes)

# data_usage	
## Les étudiants divisent l’ensemble de données en 3 ensembles (entraînement + validation + test), avec des proportions pertinentes

# data_variation	
## Les élèves introduisent certaines variations dans certaines images du jeu de données (par exemple, luminosité, facteur de zoom, translation, rotation, etc.)

# data_creation	
## Les élèves créent en groupe beaucoup d’images à l’aide d’un script afin d’augmenter la taille du jeu de données original

# data_persistency	
## Les résultats d’entraînement sont stockés (dans un fichier ou assimilés) pour permettre de relancer le binaire sans réentraîner

# tools_data	
## Les élèves utilisent efficacement au moins 3 packages différents pour la manipulation et la représentation des données (et ils peuvent l’expliquer)

# tools_ai
## Les étudiants utilisent efficacement tout logiciel d’apprentissage automatique pour des procédures algorithmiques (et ils peuvent l’expliquer)

# tools_justify	
## Les étudiants peuvent justifier, de manière professionnelle, pourquoi ils choisissent un forfait plutôt qu’un autre

# proc_preprocess	
## Les étudiants pré-traitent les données brutes, incluant au moins le profilage des données ET le nettoyage des données

# proc_tvt
## Les élèves appliquent une procédure de test de validation de train (et ils peuvent l’expliquer)

# proc_cv
## Les élèves appliquent une procédure de validation croisée et comparent ses résultats avec une répartition train-test (et ils peuvent l’expliquer)

# proc_speed	
## Les élèves normalisent les images lors du prétraitement afin d’accélérer le processus d’entraînement

# proc_pipelines	
## Les étudiants utilisent des pipelines pour organiser, garantir la fiabilité et automatiser l’ensemble du flux de traitement des données et d’entraînement des modèles

# algo_exploration	
## Les élèves explorent et testent diverses méthodes d’apprentissage automatique et comparent leurs résultats

# algo_metrics	
## Le choix des étudiants pour un algorithme adapté dépend des métriques d’évaluation appropriées

# algo_tuning	
## Les élèves ajustent les paramètres de tout modèle d’apprentissage automatique (et ils peuvent les expliquer)

# algo_reduction
## Les élèves appliquent une technique de dimensions réduites (LDA, PCA, t-SNE, etc.)

# algo_optimization	
## Les étudiants mettent en œuvre certaines techniques d’optimisation (comme l’ajustement des hyperparamètres) et l’ingénierie des fonctionnalités efficace

# versioning_basics	
## Les élèves utilisent un outil de versionnement avec un flux de travail approprié, incluant des commits réguliers et atomiques avec des messages descriptifs, ainsi qu’un fichier gitignore

# presentation	
## Le projet est présenté de manière professionnelle avec un support pertinent (diapositives et/ou démonstration)

# argumentation	
## Les élèves soutiennent leurs réflexions et défendent leurs choix avec des arguments bien développés et illustrés qui mettent en avant leurs compétences analytiques
